# 🔒 ConfiSafe

Sistema acadêmico desenvolvido para **gestão e monitoramento de segurança em espaços confinados**, alinhado às diretrizes da **NR-33 (Norma Regulamentadora 33)**.

O projeto **ConfiSafe** tem como objetivo apresentar uma solução tecnológica para auxiliar empresas na organização, monitoramento e controle de operações realizadas em ambientes confinados, aumentando a segurança dos trabalhadores.

O sistema conta com **interface web (desktop)** e previsão de **expansão para aplicação mobile**, permitindo acesso facilitado às funcionalidades da plataforma.

# 🎯 Objetivo Geral

Desenvolver uma solução digital capaz de auxiliar no **gerenciamento de segurança em espaços confinados**, permitindo o controle de acessos, monitoramento de operações, organização de treinamentos e gestão de equipamentos de proteção, seguindo as diretrizes da **NR-33**.

---

# 📌 Objetivos Específicos

- Criar uma **vitrine digital (Landing Page)** para apresentação da plataforma.
- Desenvolver uma interface intuitiva para acesso ao sistema.
- Implementar telas de **login e cadastro de usuários**.
- Apresentar os principais módulos do sistema.
- Estruturar o projeto para futura integração com **Back-end e banco de dados**.
- Facilitar o acesso ao sistema através de **interface web e futura aplicação mobile**.
- Demonstrar a proposta do sistema no contexto acadêmico do curso de **Análise e Desenvolvimento de Sistemas (ADS)**.

# 💻 Front-end

O **Front-end** é responsável pela construção da interface visual do sistema e pela interação do usuário com a aplicação.

Nesta fase foi desenvolvida a **Landing Page (Vitrine Digital)** do sistema CONFISAFE.

### Funcionalidades da Vitrine Digital

- Apresentação do sistema
- Explicação da proposta da plataforma
- Destaque dos principais módulos
- Acesso às páginas de **Login** e **Cadastro**

### Tecnologias utilizadas

- **HTML5**
- **CSS3**
- **JavaScript**

Essas tecnologias foram utilizadas para garantir uma interface **moderna, responsiva e acessível**.

---

# ⚙️ Back-end

O **Back-end** será responsável pela lógica de funcionamento do sistema, comunicação com banco de dados e gerenciamento das informações da plataforma.
